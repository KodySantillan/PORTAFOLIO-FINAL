Resumen de la semana
¿Qué vimos esta semana?

La evaluación y optimización de la arquitectura de software es una fase fundamental para verificar que la arquitectura cumple con los requisitos funcionales y no funcionales, detectar problemas y aplicar mejoras.

6 criterios de calidad evaluados: Rendimiento (tiempo de respuesta, CPU, memoria), Escalabilidad (vertical y horizontal con microservicios y Docker), Seguridad (autenticación, autorización, HTTPS, CSRF), Mantenibilidad (modularidad, principios SOLID), Disponibilidad (servidores redundantes, monitoreo continuo) e Interoperabilidad (APIs REST, JSON/XML).

6 tipos de pruebas: Funcionales, rendimiento, carga, estrés, seguridad (OWASP ZAP, Burp Suite) e integración.

Estrategias de optimización: Código (SOLID, eliminar duplicados), base de datos (índices, caché), infraestructura (Docker, Kubernetes, balanceadores) y comunicación (compresión, APIs REST eficientes).

Proceso de evaluación en 10 fases: Analizar arquitectura → seleccionar framework → configurar entorno → crear estructura → implementar componentes → aplicar patrones → integrar módulos → realizar pruebas → desplegar → mantenimiento continuo.
